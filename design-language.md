# Notion Design Language (with Custom Accent System)

## 1. Core Philosophy

This design system is based on Notion’s approach:

- Clarity over decoration  
- Function over noise  
- Flexibility over rigid structure  

With one added principle:

> A single accent colour can guide attention — without breaking calm minimalism.

---

## 2. Visual Style

### Minimal Base
- White / off-white background
- High-contrast black text
- Soft grey UI elements
- No heavy styling or gradients

### Accent Colour (Primary Customisation)
**#CB5928**

Used sparingly to highlight importance, not decorate.

---

## 3. How the Accent Colour is Used

The accent colour is **functional, not decorative**.

### Allowed uses:
- Active states (selected tabs, toggles)
- Links and interactive highlights
- Key buttons (primary CTA only)
- Important tags (e.g. “Urgent”, “Live”, “New”)
- Focus states (input fields, selected blocks)

### NOT used for:
- Large backgrounds
- Paragraph text
- Full sections
- Decorative illustrations
- Overuse in UI components

---

## 4. Colour System

### Base Palette
- White: #FFFFFF  
- Background grey: #F7F7F5  
- Light border: #E9E9E7  

### Text
- Primary: #111111  
- Secondary: #666666  
- Muted: #999999  

### Accent
- Primary Accent: **#CB5928**
- Hover Accent: slightly darker variation of #CB5928
- Soft Accent Background: very light tint (5–10% opacity)

---

## 5. Typography

- Clean sans-serif (Inter-style)
- Strong hierarchy but minimal variation

### Hierarchy:
- H1: bold, large, black
- H2: bold, medium
- Body: regular weight
- Caption: muted grey

No decorative fonts. No stylistic variation.

---

## 6. Layout System

### Block-Based Structure (Core Concept)
Everything is built from blocks:
- Text
- Images
- Databases
- Toggles
- Lists

Blocks behave like LEGO:
- Move freely
- Stack vertically
- Nest inside each other

---

## 7. Components

### Buttons
- Default: light grey background
- Primary: #CB5928 fill
- Text always white on primary buttons
- Rounded corners (soft, not pill)

### Links
- Default text: black
- Hover: #CB5928 underline or highlight

### Cards
- White background
- Subtle border
- Accent used only for labels or indicators

---

## 8. Interaction Style

- Subtle transitions (fast, not flashy)
- No heavy animation
- Hover states are minimal (slight colour shift or underline)
- Focus states use accent colour clearly but lightly

---

## 9. Illustration Style (Notioly-inspired)

- Black & white line illustrations
- Human-centred, everyday actions
- Soft humour, warm tone
- Used for onboarding, empty states, and explanation screens

Accent colour can be used ONLY for:
- Small highlights (e.g. a folder, note, or object detail)
- Never full illustration colouring

---

## 10. UI Behaviour Principles

- Everything should feel “quiet”
- Accent colour should guide attention, not demand it
- Space is a design tool (not filled areas)
- Content always leads, UI supports

---

## 11. Tone of Voice

- Simple language
- No jargon
- Calm and instructional
- Friendly but not overly enthusiastic

Example:
- “Create a new page”  
- “Add a task”  
- “Share with your team”

---

## 12. Overall Feel

A system that feels like:

> A calm, minimal workspace where everything is organised clearly — with a subtle warm accent (#CB5928) guiding attention without breaking the simplicity.
